a micro js charting framework

grapher.js is a micro (very small) pure javascript framework for graphing various charts. The goal is to be a self contained pure javascript library, that does not use canvas or require an external library, is blazing fast, with a tiny footprint and a pretty face. There are some awesome things about grapher besides its holistic approach to size:

. requires no external libraries(no jquery), no images, or css
. does NOT use canvas, no VML or SVG here.
. a HTML4 and CSS2 rendering engine.
. can be completely inlined (tiny)!
. is blazing fast!
. works practically everywhere!
. features easy to use syntax.
. is FREE , OPENSOURCE , and EVOLVING!

Only a few restrictions on how you can use grapher.js, but if you do use it, please drop me a line!

grapher.js is developed by Derek Anderson. And is inspired by Thomas Fuchs, Walter Zorn and Balamurugan S.

How it works: This library has a custom rendering engine that is based on Walter Zorn's amazingly awesome jsgraphics, with some borrowed math. The rendering engine builds all graphics in pure DIVs, and relies on the CSS2 clip property.

Bar Graph, Horizontal Style, with Bar stack

 
 
 

usage

 
<script type="text/javascript"> 
	data = {"1":[100,50,20],"2":[98],"3":[75,40],"4":[60],"5":[70,36],"6":[30]}
	options =   {"borderColor":"#000",
    "width":"300px",
    "height":"150px",
    "ymax":800,
    "gridlines":true,
	"yunits":["100","90","80","70","60","50","40","30","20","10"],
	"xunits":["Mon","Tue","Wed","Thu","Fri","Mon","Tue","Wed","Thu","Fri"],
	"style":"h"
    }
	grapher.initialize("test",options);
	grapher.bar("test",data,options);
</script>
